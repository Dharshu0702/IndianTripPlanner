# India Smart Trip Planner

AI-powered travel planning web application for destinations across India. Generate personalized trip plans with real hotel data, accurate cost breakdowns, and day-wise itineraries вЂ” all powered by a local LLM.

https://drive.google.com/file/d/1AN02h5V8d63YZznRjxDdMI7yHXLzSNow/view?usp=sharing

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React.js + Vite |
| Backend | Python Flask (REST API) |
| Database | MongoDB (PyMongo) |
| AI / LLM | Ollama (Gemma 2B, local inference) |
| Auth | JWT (Flask-JWT-Extended) |
| Routing & Maps | OSRM (Open Source Routing Machine) + Haversine fallback |
| Hotel Discovery | OpenStreetMap Nominatim API (web crawling) |
| Destination Data | Wikipedia scraping (BeautifulSoup) |
| Email | Flask-Mail (SMTP) |
| PDF | ReportLab |

## Prerequisites

- **Node.js** >= 18
- **Python** >= 3.10
- **MongoDB** running locally or Atlas URI
- **Ollama** installed with `gemma:2b` model pulled

## Quick Start

### 1. Backend Setup

```bash
cd backend

# Create virtual environment
python -m venv venv
venv\Scripts\activate  # Windows
# source venv/bin/activate  # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your settings (MongoDB URI, SMTP credentials)

# Seed destinations database
python -m app.seed.seed_destinations

# Start Flask server
python run.py
```
Backend runs at `http://localhost:5000`

### 2. Frontend Setup

```bash
cd frontend

# Install dependencies
npm install

# Start dev server
npm run dev
```
Frontend runs at `http://localhost:5173` (proxied API calls to Flask)

### 3. Ollama Setup

```bash
# Install Ollama from https://ollama.ai
ollama pull gemma:2b
ollama serve
```

> **Note:** The app works without Ollama вЂ” it will fall back to template-based trip plans with accurate costs generated by the built-in cost engine.

## Architecture

### How Gemma LLM is Connected

1. The Flask backend sends a structured prompt to **Ollama's** `/api/generate` endpoint
2. **Gemma 2B** generates 3 JSON trip plans with itineraries and activities
3. A **post-processing pipeline** validates and fixes the AI output:
   - Hotel names are checked against price tiers
   - All costs are overridden by the deterministic **cost engine** (AI costs are unreliable)
   - Itinerary day count is enforced
4. Results are **cached in MongoDB** with a 24-hour TTL (SHA-256 cache key)
5. If Ollama is unavailable, **template-based fallback plans** are generated automatically

### Web Crawling

| Source | API | Purpose |
|--------|-----|---------|
| OpenStreetMap Nominatim | `nominatim.openstreetmap.org/search` | Discover real hotels near destinations |
| Wikipedia | `en.wikipedia.org/wiki/{destination}` | Scrape destination descriptions |
| OSRM | `router.project-osrm.org/route/v1/driving/` | Get real road distance & driving time |

All crawled data is cached in MongoDB to minimize external API calls.

### MongoDB Collections

| Collection | Purpose | TTL |
