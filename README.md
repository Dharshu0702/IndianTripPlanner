# 🇮🇳 India Smart Trip Planner

AI-powered travel planning web application for destinations across India. Generate personalized trip plans with real hotel data, accurate cost breakdowns, and day-wise itineraries — all powered by a local LLM.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | React.js + Vite |
| Backend | Python Flask (REST API) |
| Database | MongoDB (PyMongo) |
| AI / LLM | Ollama (Gemma 2B, local inference) |
| Auth | JWT (Flask-JWT-Extended) |
| Routing & Maps | OSRM (Open Source Routing Machine) + Haversine fallback |
| Hotel Discovery | OpenStreetMap Nominatim API (web crawling) |
| Destination Data | Wikipedia scraping (BeautifulSoup) |
| Email | Flask-Mail (SMTP) |
| PDF | ReportLab |

## Prerequisites

- **Node.js** >= 18
- **Python** >= 3.10
- **MongoDB** running locally or Atlas URI
- **Ollama** installed with `gemma:2b` model pulled

## Quick Start

### 1. Backend Setup

```bash
cd backend

# Create virtual environment
python -m venv venv
venv\Scripts\activate  # Windows
# source venv/bin/activate  # macOS/Linux

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your settings (MongoDB URI, SMTP credentials)

# Seed destinations database
python -m app.seed.seed_destinations

# Start Flask server
python run.py
```
Backend runs at `http://localhost:5000`

### 2. Frontend Setup

```bash
cd frontend

# Install dependencies
npm install

# Start dev server
npm run dev
```
Frontend runs at `http://localhost:5173` (proxied API calls to Flask)

### 3. Ollama Setup

```bash
# Install Ollama from https://ollama.ai
ollama pull gemma:2b
ollama serve
```

> **Note:** The app works without Ollama — it will fall back to template-based trip plans with accurate costs generated by the built-in cost engine.

## Features

### Core
- 🤖 **AI-generated travel plans** — Gemma 2B generates 3 plans (Budget / Balanced / Luxury) with day-wise itineraries
- 💰 **Deterministic cost engine** — Accurate ₹ INR pricing for transport, hotels, food, local travel, and activities
- ✏️ **Plan customization** — Modify travelers, days, transport mode with live cost recalculation
- 🏨 **Real hotel discovery** — Web crawling via OpenStreetMap Nominatim API, categorized by tier

### Data & Maps
- 🏛️ **70+ curated destinations** across 28 Indian states
- 📍 **Browser geolocation** with manual address fallback
- 🗺️ **OSRM road routing** — Real road distance and driving time (Haversine fallback)
- 🔗 **OpenStreetMap route links** for each trip
- 📝 **Wikipedia-enriched descriptions** — Scraped destination info via BeautifulSoup

### Platform
- 🔐 **JWT authentication** — User and Admin roles with protected routes
- 📧 **Email notifications** — Booking confirmation and admin alerts via SMTP
- 📊 **Admin dashboard** — View all bookings, approve/reject requests, dashboard stats
- 📄 **PDF export** — Download trip itineraries and booking confirmations

## Architecture

### How Gemma LLM is Connected

1. The Flask backend sends a structured prompt to **Ollama's** `/api/generate` endpoint
2. **Gemma 2B** generates 3 JSON trip plans with itineraries and activities
3. A **post-processing pipeline** validates and fixes the AI output:
   - Hotel names are checked against price tiers
   - All costs are overridden by the deterministic **cost engine** (AI costs are unreliable)
   - Itinerary day count is enforced
4. Results are **cached in MongoDB** with a 24-hour TTL (SHA-256 cache key)
5. If Ollama is unavailable, **template-based fallback plans** are generated automatically

### Web Crawling

| Source | API | Purpose |
|--------|-----|---------|
| OpenStreetMap Nominatim | `nominatim.openstreetmap.org/search` | Discover real hotels near destinations |
| Wikipedia | `en.wikipedia.org/wiki/{destination}` | Scrape destination descriptions |
| OSRM | `router.project-osrm.org/route/v1/driving/` | Get real road distance & driving time |

All crawled data is cached in MongoDB to minimize external API calls.

### MongoDB Collections

| Collection | Purpose | TTL |
|-----------|---------|-----|
| `users` | User accounts (email, password hash, role) | — |
| `trips` | Generated trip plans per user | — |
| `bookings` | Booking requests and approval status | — |
| `destinations` | Seeded destinations with scraped descriptions | — |
| `ai_cache` | Cached LLM responses | 24h |
| `hotel_cache` | Cached hotel scraping results | 24h |
| `distance_cache` | Cached OSRM distance results | — |

## API Endpoints

### Auth
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/auth/signup` | POST | Register user |
| `/api/auth/login` | POST | Login |
| `/api/auth/me` | GET | Current user profile |

### Destinations
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/destinations/states` | GET | All states |
| `/api/destinations/by-state/:state` | GET | Destinations for a state |

### Trips
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/trips/generate` | POST | Generate 3 AI trip plans |
| `/api/trips/my-trips` | GET | User's saved trips |
| `/api/trips/:tripId` | GET | Get specific trip |
| `/api/trips/recalculate` | POST | Recalculate cost with custom parameters |

### Hotels
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/hotels` | GET | Scrape real hotels for a destination and tier |

### Bookings
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/bookings` | POST | Create booking |
| `/api/bookings/my-bookings` | GET | User's bookings |

### Admin
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/admin/stats` | GET | Dashboard statistics |
| `/api/admin/bookings` | GET | All bookings |
| `/api/admin/bookings/:id/status` | PATCH | Approve / reject booking |

### PDF
| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/pdf/trip/:id/:plan` | GET | Download plan itinerary PDF |
| `/api/pdf/booking/:id` | GET | Download booking confirmation PDF |
